Création de vos comptes de services

Un guide simple, étape par étape, pour créer les comptes qui font fonctionner votre application — paiements, SMS et e-mails sous votre propre nom.

Préparé par Achraf Benamrane · Développeur |Progix |GetDraft

Bonjour ! 👋 Pour lancer GetDraft sous votre nom, quelques services doivent fonctionner sur vos comptes — ainsi les paiements arrivent sur votre banque, les coûts de SMS et de vérification vous sont facturés à vous, et les e-mails partent de votre marque.

Pas besoin d'être technique. Suivez les étapes, copiez quelques codes, et envoyez-les-nous en sécurité (voir la dernière page). Bloqué à une étape ? Faites une capture d'écran et envoyez-la-nous — on vous indique exactement où cliquer.

⏱️ Durée totale : ~30–45 minutes ☕ Astuce : faites-les un par un
💡 Le plus simple : si vous préférez, invitez-nous comme membre sur chaque compte (Stripe, Twilio…) et nous configurons tout à votre place — vous n'avez alors rien à copier. Sinon, suivez les étapes ci-dessous.
📌 Avant de commencer : créez chaque compte avec l'e-mail de votre entreprise (et non personnel), pour que l'entreprise soit propriétaire de tout. Utilisez un mot de passe fort et gardez-le.

Les 4 comptes nécessaires

1

💳 Stripe — pour encaisser les paiements

À quoi ça sert : permet aux utilisateurs de payer les abonnements et les swipes. L'argent va directement sur votre compte bancaire.

  1. Allez sur Créer un compte Stripe → e-mail, nom, mot de passe → confirmez l'e-mail reçu.
  2. Renseignez vos informations d'entreprise et votre compte bancaire (c'est ainsi que vous serez payé). Vous pouvez le faire plus tard — les tests fonctionnent sans.
  3. Récupérez vos clés ici 👉 Développeurs → Clés API : copiez la Clé publiable (pk_…) et cliquez sur « Révéler » pour copier la Clé secrète (sk_…).
  4. Créez vos forfaits ici 👉 Catalogue de produits → + Ajouter un produit → un par forfait (ex. Starter, Pro) avec un prix → Enregistrer. Ouvrez chacun et copiez son ID de prix (price_…).
  5. Webhook : 👉 Développeurs → Webhooks → Ajouter un endpoint → collez cette URL : https://getdraft-api-production.up.railway.app/api/webhooks/stripe → cochez « tous les évènements » → Créer → copiez le « Signing secret » (whsec_…).
📩 À nous envoyer : Clé publiable (pk_…), Clé secrète (sk_…), les ID de prix (price_…), et le Signing secret du webhook (whsec_…).
2

📱 Twilio — pour envoyer les codes SMS

À quoi ça sert : à l'inscription par téléphone, Twilio envoie le code de vérification par SMS. Le petit coût par message vous est facturé.

  1. Allez sur S'inscrire sur Twilio → créez le compte → vérifiez votre e-mail et votre téléphone.
  2. Passez du compte d'essai au compte payant (ajoutez un moyen de paiement). ⚠️ Important — en essai, Twilio n'envoie de SMS qu'aux numéros vérifiés ; le compte payant permet d'envoyer à tous vos utilisateurs.
  3. Sur le tableau de bord Twilio, dans l'encart « Account Info », copiez le Account SID (AC…) et l'Auth Token (cliquez pour révéler).
  4. Créez un service de vérification 👉 Verify → Services → Create new → nom « GetDraft » → copiez le Service SID (VA…).
📩 À nous envoyer : Account SID (AC…), Auth Token, et Verify Service SID (VA…).
3

✉️ Resend — pour envoyer les e-mails

À quoi ça sert : envoie les codes par e-mail et les notifications depuis votre domaine (ex. noreply@votreentreprise.com), pour un rendu pro qui arrive en boîte de réception.

  1. Allez sur S'inscrire sur Resend et créez votre compte.
  2. Ajoutez votre domaine 👉 Domains → Add Domain → saisissez votre domaine. Resend affiche des enregistrements DNS.
    🛠️ C'est la seule étape un peu technique — envoyez-nous une capture de ces enregistrements (ou donnez l'accès à votre responsable site/IT) et on vous aide à les ajouter. ~5 minutes.
  3. Créez la clé 👉 API Keys → Create API Key → nom « GetDraft » → copiez-la (re_…).
📩 À nous envoyer : la clé (re_…), votre domaine, et l'adresse d'expéditeur souhaitée (ex. GetDraft <noreply@votreentreprise.com>).
4

🪪 Didit — pour vérifier l'identité (KYC)

À quoi ça sert : vérifie la pièce d'identité + le visage de l'utilisateur à l'inscription, pour garantir de vraies personnes.

  1. Allez sur Créer un compte entreprise Didit.
  2. Dans le tableau de bord, ouvrez Workflows (ou « Verification ») → repérez le workflow KYC → copiez son Workflow ID.
  3. Ouvrez Settings / API → copiez votre API Key.
  4. Ouvrez Webhooks → ajoutez l'URL https://getdraft-api-production.up.railway.app/api/kyc/webhooks/didit → copiez le Webhook Secret.
💡 Les menus Didit peuvent varier — si vous ne trouvez pas l'un des trois, envoyez une capture de votre tableau de bord et on vous indique l'endroit exact.
📩 À nous envoyer : l'API Key, le Workflow ID, et le Webhook Secret.

Optionnel — au moment de publier

📲 Comptes de stores

Pas des « clés », mais nécessaires pour publier l'app sous le nom de votre entreprise. À faire plus tard.

  • Google Play (Android) — 25 $ une fois → Inscription Google Play Console
  • Apple (iPhone) — 99 $/an → Apple Developer Program

À lire — envoyer les clés en sécurité

🔒 Comment nous envoyer les clés en sécurité

Ces clés sont comme des mots de passe de votre argent et de vos comptes — ne les collez jamais dans un message WhatsApp ou e-mail normal.

  • Le plus simple : créez un compte gratuit Bitwarden → créez une Note sécurisée → collez toutes les clés → utilisez « Send » pour nous la partager.
  • Ou via un lien sécurisé qu'on met en place ensemble — demandez-nous.
  • ⚠️ Si vous partagez une clé par erreur dans un chat normal, dites-le-nous — on la régénère (c'est rapide).

Récapitulatif — à nous envoyer

Stripe : Clé publiable (pk_…), Clé secrète (sk_…), ID de prix (price_…), Webhook secret (whsec_…)
Twilio : Account SID (AC…), Auth Token, Verify Service SID (VA…)
Resend : clé (re_…), votre domaine, votre e-mail d'expéditeur
Didit : API Key, Workflow ID, Webhook Secret
Plus tard : comptes Google Play + Apple Developer

Une fois reçues, GetDraft tourne 100% sur vos comptes — vos paiements, vos messages, votre marque. 🚀
